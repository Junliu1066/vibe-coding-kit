# 项目说明书

## 一句话
文件整理大师 —— 一键整理混乱文件夹的跨平台桌面工具（分类+去重+重命名三合一）。

## 需求基准描述（来自 vibe-coding-requirements）
**场景：** 普通电脑用户的下载文件夹、桌面常年堆积数百个文件，图片文档视频混在一起，找文件靠搜索，空间被重复文件浪费。
**目标：** 提供一个傻瓜式桌面工具，用户选中文件夹后一键自动分类归档、找出重复文件释放空间、批量规范文件命名。
**约束：** 单人开发，一周出 MVP，Flutter 技术栈，Mac+Windows 双平台，一次性买断模式。
**验收：** 用户能在 3 步内完成一次完整整理（选文件夹→预览→确认执行），所有删除操作可回滚。

## 产品思考模块（预留）
- 本模块先作为后续对齐入口，不替代 PRD，不作为当前开发出口门。
- 后续可逐步沉淀：用户价值、真实使用场景、替代方案、差异化、商业/增长、优先级取舍、产品风险。

## 选定技术栈（来自 vibe-coding-architecture）
| 项 | 选定 | 否决方案及理由 |
|----|------|--------------|
| 框架 | Flutter Desktop | Electron：体积大（~150MB+），内存占用高；Tauri：Rust 学习成本高，单人一周不现实；Qt：C++ 开发效率低，跨平台 UI 一致性差 |
| 语言 | Dart | — |
| 本地存储 | SQLite (sqflite) | JSON 文件：并发写不安全；Hive：操作历史需要关系查询 |
| 状态管理 | Riverpod | BLoC：模板代码太多，小项目杀鸡用牛刀；Provider：已落后，Riverpod 是其继任者 |
| 激活方案 | 本地密钥校验 + 服务器验证 | 纯在线激活：离线场景不可用；纯本地：容易被破解 |
| 打包 | flutter_distributor | Flutter 官方打包工具功能有限 |

## 开发规范要点
- 分支策略：main 分支即发布分支，feature 分支开发，PR 合并
- 日志：使用 `logger` 包，分级（debug/info/warning/error），不记录文件内容
- 错误处理：引擎层抛自定义异常，UI 层统一捕获并展示用户友好提示
- 代码风格：遵循 `flutter_lints` 推荐规则
- 注释：简体中文

## 部署步骤
1. `flutter pub get` 安装依赖
2. `flutter run -d macos` / `flutter run -d windows` 本地调试
3. `flutter_distributor package --platform macos` 打包 macOS
4. `flutter_distributor package --platform windows` 打包 Windows
5. 上传安装包到服务器，配置下载页面
6. 版本号写入服务器 API，客户端检查更新

## 决策复盘 / 案例库
- 2026-07-01｜项目进度卡模块抽象
  - 问题：原有 `docs/进度账本.md` 能约束 S1/S2/S3 流程，但开发期任务一多，AI 恢复上下文容易默认读取大量历史、模块细节和任务记录。
  - 可选方案：把详细进度继续塞进 `docs/进度账本.md`；或单独建立 `.dsu/progress/`，用总表、模块卡、任务卡分层承载。
  - 最终决定：新增通用项目进度卡模块，固定目录为 `.dsu/progress/`。默认只读 `index.md`，按需下钻模块卡，复盘时才读任务卡。
  - 代价：多维护一组三层文档；每个任务结束后必须同步 index、模块卡、任务卡，否则会产生状态漂移。
  - 下次复用规则：流程门继续归 `docs/进度账本.md` 管；开发期上下文恢复归 `.dsu/progress/` 管。总表只写索引级信息，不写模块实现细节。
