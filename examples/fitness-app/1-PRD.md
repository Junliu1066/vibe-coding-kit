# 产品需求文档（PRD）—— 燃动 FitPulse

## 一句话

**三合一轻量健身伴侣**：跑步节拍器 + 组间歇计时 + 喝水提醒，免费、轻量、不打扰。

---

## 背景与问题

市面上的健身 APP 要么大而全（Keep、咕咚）臃肿复杂，要么只做单一功能（独立节拍器、独立间歇计时器、独立喝水提醒 APP）。**没有一个 APP 把跑步节拍器、组间歇计时、喝水提醒三样干净地整合在一起。** 用户想在跑步、力量训练、日常补水之间无缝切换，需要装三个 APP 来回切，体验割裂。

本产品定位：**极简三合一**——打开即用，免费，不搞社交、不搞排行榜、不搞打卡焦虑。

---

## 目标用户与场景

| 维度 | 说明 |
|------|------|
| 用户群体 | 跑步爱好者、力量训练者、HIIT 训练者、需要提醒喝水的普通用户 |
| 设备 | 手机（PWA，可装到桌面，安卓/iOS 通用） |
| 使用场景 | 户外跑步时打开节拍器控步频；健身房练力量/有氧时用间歇计时；日常办公/生活中收到喝水提醒 |

---

## 产品目标与成功指标

- **目标**：验证"三合一简洁健身伴侣"是否有人需要
- **成功信号**：找 10 个真实用户试用，过半（≥6 人）表示"好用、还想用"
- **验证入口**：demo 中内嵌反馈收集入口。首页右下角悬浮按钮"💬 反馈 / 提建议"，点击弹出表单（评价 + 联系方式选填）。提交后数据存 IndexedDB `feedback` 表，用户可在浏览器中查看（暂不上传云端）。验证阶段由产品方定期导出反馈数据进行分析

### 跑步节拍器页面

```
┌──────────────────────┐
│  ← 返回    跑步节拍器  │
├──────────────────────┤
│                      │
│      170 BPM         │  ← 大号数字显示当前 BPM
│    ───●───           │  ← 滑块调整（范围 60-240）
│                      │
│  预设：160 │ 170 │ 180│  ← 快速切换预设
│                      │
│  节奏音：滴声 │ 鼓点 │ 人声 │ ← 音色选择
│                      │
│    ┌──────────┐      │
│    │   ▶ 开始  │      │  ← 主按钮
│    └──────────┘      │
│                      │
│  上次用时：12:34      │  ← 可选显示
└──────────────────────┘
```

- **滑块**：拖拽或点击 +/- 调整 BPM，实时显示数字。**与预设按钮的联动规则**：用户拖滑块到任意值（如 175）→ 所有预设按钮取消高亮（因为当前值不是任何预设值）；用户点某个预设按钮（如 180）→ 滑块跳到 180，该按钮高亮。即：只有当前 BPM 恰好等于某个预设值时，对应按钮才高亮。
- **预设按钮**：点击直接切换对应 BPM
- **音色切换**：三选一（滴声/鼓点/人声），选中态高亮
  - **滴声**：类似电子节拍器的短促"滴"声，使用 Web Audio API OscillatorNode 生成（正弦波，频率 880Hz，持续 50ms）
  - **鼓点**：模拟打击乐短促"咚"声，使用 OscillatorNode 生成（正弦波，频率 150Hz，持续 80ms，带衰减包络）
  - **人声**：播报数字节拍（如"1,2,3,4"），帮用户默数步频。如加载失败自动降级为滴声并提示
- **开始/停止**：点击后开始播放节奏音，按钮变为"■ 停止"，停止时记录本次时长
- **空态**：首次进入（IndexedDB 无记录），BPM 默认 170，音色默认"滴声"；非首次进入则恢复上次保存的值
- **加载态**：切换音色时短暂加载提示（人声模式需预加载资源，滴声和鼓点即时可用无需加载）
- **错误态**：音频播放失败时提示"声音播放失败，请检查手机音量设置"

### 组间歇计时页面

```
┌──────────────────────┐
│  ← 返回    组间歇计时  │
├──────────────────────┤
│                      │
│  模式：[自动循环] [手动触发] │ ← 模式切换 Tab
│                      │
│  ┌─ 当前方案 ──────┐  │
│  │ Tabata 经典      │  │  ← 可下拉切换
│  │ 练20s 歇10s ×8组 │  │
│  └────────────────┘  │
│                      │
│  ┌─ 自定义 ────────┐  │
│  │ 训练: [40] 秒    │  │  ← 可编辑
│  │ 休息: [20] 秒    │  │
│  │ 组数: [8]        │  │
│  └────────────────┘  │
│                      │
│        ⏱ 00:32       │  ← 当前倒计时（大号）
│    还剩 5 组          │  ← 剩余组数
│                      │
│    ┌──────────┐      │
│    │   ▶ 开始  │      │  ← 主按钮（手动模式下为"完成本组 ▶"）
│    └──────────┘      │
│                      │
├──────────────────────┤
│  📋 上传训练计划      │  ← AI 分析入口
│  (支持图片/文本)      │
└──────────────────────┘
```

- **模式切换**：自动循环 / 手动触发，点击切换
- **方案选择**：预置方案下拉选择，或自定义
- **自动循环模式**：用户点"开始"→ 系统自动跑完所有组。每组的流程：训练倒计时 → 语音播报"休息时间到"→ 休息倒计时 → 语音播报"开始训练"→ 进入下一组训练。全程不需要用户再碰手机。
  - **暂停行为**：用户点"停止"按钮 → 当前倒计时暂停 → 按钮变为"继续"→ 用户点"继续"→ 从暂停时的剩余秒数继续倒计时（不跳过当前组、不重置当前组）。如果用户关闭了页面或 APP，则当前进度丢失，下次打开重新开始。
- **手动触发模式**：用户做完一组动作后 → 自己点"完成本组"按钮 → 系统开始休息倒计时 → 倒计时结束语音播报"休息时间到"→ 用户自己开始下一组动作 → 再点"完成本组"→ 循环直到所有组完成。**注意**：手动模式下，用户不点"完成本组"，计时器不会自动进入休息——因为力量训练（如卧推）每组用时因人而异，由用户自己掌控节奏。
- **语音播报**：合成语音播报，不依赖外部服务（Web Speech API）
- **上传训练计划**：用户可上传图片（jpg/png，单张最大 10MB）或文本描述训练计划，点击后弹出输入方式选择：
  - 📷 **拍照/选图片**：用户拍一张手写训练计划或截图，AI 识别图片中的文字并提取参数
  - ✏️ **文字输入**：用户直接打字描述，如"我想做 Tabata，练 20 秒歇 10 秒，总共 8 组"
  - 远端 AI 解析 → 自动填入自定义区域的训练秒数、休息秒数、组数
- **空态**：首次进入（IndexedDB 无记录）默认选中"Tabata 经典"方案，自动循环模式；非首次进入恢复上次保存的模式和方案
- **加载态**：AI 分析训练计划时显示"分析中..."
- **错误态**：AI 分析失败提示"训练计划解析失败，请手动设置或重试"
- **编辑态**：自定义区域可编辑，输入非法值（负数/0）时标红提示

### 喝水提醒页面

```
┌──────────────────────┐
│  ← 返回    喝水提醒   │
├──────────────────────┤
│                      │
│   💧 下次提醒         │
│     22 分钟后         │  ← 大号倒计时
│                      │
│  提醒间隔：[45] 分钟  │  ← 可编辑
│                      │
│  ┌────────────────┐  │
│  │ 🔔 提醒已开启   │  │  ← 开关状态
│  │    点击关闭     │  │
│  └────────────────┘  │
│                      │
│  运动联动：          │
│  [✓] 运动时每 15 分钟提醒 │
│                      │
│  免打扰：22:00-08:00  │  ← 固定时段
│  [✓] 运动时忽略免打扰  │ ← 勾选开关
│                      │
│  今日已提醒：4 次     │  ← 统计
│                      │
├──────────────────────┤
│  提醒方式：震动 + 通知 │
└──────────────────────┘
```

- **间隔设置**：用户自由输入分钟数，默认 45
- **开关**：一键开启/关闭提醒
- **运动联动**：跑步节拍器或间歇计时任一处于"运行中"状态时，将喝水提醒间隔调整为 `min(用户设定值, 15)` 分钟。即：如果用户设了 5 分钟，运动时保持 5 分钟（不升到 15）；如果用户设了 45 分钟，运动时缩短为 15 分钟。两者都停止后，恢复用户设定的原始间隔。
  - **判定方式**：`metronome.running === true` 或 `interval.running === true` → 启用 15 分钟间隔
  - **切换时机**：运动开始时立即重新计算下次提醒时间（如当前距上次提醒已超过 15 分钟，则立即触发一次）
- **统计**：今日提醒次数（当天 00:00 归零。判定方式：每次触发提醒前，比较 `lastReminderDate` 与当前日期，跨日则 `todayCount` 置 0）
- **提醒方式**：通知栏推送 + 震动（PWA Notification API + Vibration API）。**免打扰时段**：22:00-08:00 不推送通知（仅静默记录提醒次数），用户可在设置中关闭免打扰限制。运动期间忽略免打扰（跑步/训练时正常提醒）
- **空态**：首次进入（IndexedDB 无记录）默认 45 分钟间隔，提醒已开启，运动联动已开启；非首次进入恢复上次保存的值
- **错误态**：通知权限被拒绝时提示"通知权限未开启，请到手机设置中允许通知"

### 全局状态覆盖

**三个功能可同时独立运行**：用户可以一边开着跑步节拍器（听节奏），一边让间歇计时跑着（练 HIIT），同时喝水提醒在后台按时触发。三个功能的状态（running/enabled）互不阻塞，各自独立。运动联动检测任意一个"运动中"即可触发。

**页面导航不影响功能运行**：用户从功能页返回首页时，正在运行的功能**继续在后台运行**（节拍器继续响、间歇计时继续倒计时、喝水提醒继续定时）。用户再次进入该功能页时，看到的是实时状态。**唯一例外**：关闭 APP/浏览器标签页则所有运行中的功能停止（PWA 后台限制）。

| 状态 | 表现 |
|------|------|
| **空态** | 首次打开 APP，三个功能均无历史数据，显示默认值和引导文案 |
| **加载态** | AI 分析训练计划时显示加载动画，节拍器切换音色时预加载 |
| **错误态** | 音频失败、通知权限拒绝、AI 分析失败——均给出中文提示 + 下一步操作建议 |
| **编辑态** | 所有可编辑字段（BPM、训练秒数、喝水间隔等）点击后可修改，非法值标红 |
| **移动端** | 全移动端优先设计，触控目标 ≥ 44px，卡片全宽 |

## 核心功能的处理规则

### 跑步节拍器处理规则

| 规则 | 具体实现 |
|------|----------|
| BPM 范围 | 60-240，滑块自动钳制到边界值 |
| BPM 默认值 | **首次使用**（IndexedDB 中无记录时）默认 170 BPM。之后每次打开 APP，恢复用户上次设置的 BPM 值（从 IndexedDB 读取）。用户调过 BPM 后自动保存，无需手动"保存" |
| 节奏音生成 | 使用 Web Audio API 生成滴声/鼓点声；人声模式播报数字节拍（具体实现见开发文档） |
| 音色切换 | 切换时停止当前播放，预加载新音色，再开始 |
| 精度 | 节拍间隔 = 60000 / BPM 毫秒，误差控制在 ±5ms 内 |
| 非法输入 | BPM 通过滑块和预设按钮设置（无文本输入框），滑块自动钳制到 60-240 范围，不存在非法输入场景 |

### 组间歇计时处理规则

| 规则 | 具体实现 |
|------|----------|
| 预置方案 | Tabata 经典（练 20s 歇 10s ×8 组，共 4 分钟，适合燃脂冲刺）、HIIT 燃脂（练 40s 歇 20s ×6 组，共 6 分钟，适合高强度间歇）、力量增肌（练 45s 歇 60s ×4 组，共 7 分钟，适合卧推/深蹲等力量动作，建议配合手动触发模式）、核心训练（练 30s 歇 15s ×5 组，共 3 分 45 秒，适合卷腹/平板支撑） |
| 自定义方案校验 | 训练/休息秒数：1-600；组数：1-99。非法值（空值、负数、0、超范围、非数字）标红并阻止开始。点击"开始"时逐字段校验，第一个非法字段自动聚焦并提示具体错误（如"训练秒数请输入 1-600 之间的数字"） |
| 自动循环 | 训练→语音"开始训练"→倒计时→语音"休息时间到"→休息→语音"休息结束，开始训练"→循环 |
| 手动触发 | 用户点"完成本组"→休息倒计时→语音"休息时间到"→等待用户开始下一组 |
| 语音播报 | 使用 Web Speech API（SpeechSynthesis），无网络依赖。**静音处理**：如果浏览器无法发出声音（如手机静音开关开启、音量=0），语音播报可能听不到——此时用震动（Vibration API，200ms × 2 次）作为替代信号。训练→休息切换时震动 2 次，休息→训练切换时震动 1 次。用户可通过震动模式区分当前阶段 |
| 后台运行 | 关闭 APP/标签页则计时中断，页面内计时器通过浏览器 API 保持运行 |

### 喝水提醒处理规则

| 规则 | 具体实现 |
|------|----------|
| 间隔范围 | 1-480 分钟（1 分钟到 8 小时），非法值标红 |
| 默认间隔 | 45 分钟 |
| 运动联动 | 节拍器或间歇计时运行时，自动将间隔缩短为 15 分钟；停止运动后恢复原间隔 |
| 提醒方式 | Notification API 弹通知 + Vibration API 震动（200ms） |
| 通知权限 | 首次使用时请求通知权限；拒绝后在设置页面显示"权限未开启"提示 |
| 去重 | 同一分钟内不重复提醒。通知发出后即计为"已提醒"（`todayCount + 1`），无论用户是否点击/划掉通知。如果用户连续 3 次未与通知交互（既未点击也未打开 APP），第 4 次提醒时将间隔延长一倍（如从 45 分钟变为 90 分钟），避免骚扰。用户打开 APP 后恢复原间隔 |

### AI 训练计划解析（P1）

| 规则 | 具体实现 |
|------|----------|
| 输入格式 | 支持图片（jpg/png，单张最大 10MB，前端压缩至 1024px 宽）和文本输入（最大 500 字） |
| 输入示例 | 文本："做个 40 秒的 HIIT，休息 20 秒，做 6 轮"；图片：手写训练计划照片 |
| 解析目标 | 从用户描述中提取：训练秒数、休息秒数、组数 |
| 处理策略 | 调用远端 AI API（如 Claude / GPT），prompt 中明确要求返回结构化 JSON |
| Prompt 模板 | "请从以下训练计划描述中提取：训练秒数、休息秒数、组数、方案名称。以 JSON 格式返回，字段名为 training_seconds, rest_seconds, rounds, name。如果某个字段无法提取，用 null。仅返回 JSON，不要其他文字。\n\n用户描述：{user_input}" |
| 输出格式 | `{"training_seconds": 40, "rest_seconds": 20, "rounds": 8, "name": "自定义方案"}` |
| 缺失处理 | 任一字段提取不到时，使用默认值（训练 40s、休息 20s、8 组）并提示用户"部分字段未识别，已用默认值填充，请检查" |
| 解析失败 | 提示"训练计划解析失败，请手动设置或重试" |

## 出错和边界

| 场景 | 提示文案 | 系统行为 |
|------|----------|----------|
| 音频播放失败 | "声音播放失败，请检查手机音量设置" | 停止节拍器，按钮恢复为"开始" |
| 通知权限被拒绝 | "通知权限未开启，请到手机设置中允许通知，否则收不到喝水提醒哦" | 提醒功能仍可开启，但通知不会弹出 |
| AI 解析失败 | "训练计划解析失败，请手动设置或重试" | 保留用户上传的内容，允许重试 |
| AI 解析超时 | "分析超时，请检查网络后重试" | 超时时间 30 秒 |
| 输入非法值 | 输入框标红 + 下方显示提示"请输入 X-Y 之间的数字" | 阻止开始操作 |
| IndexedDB 不可用 | "数据存储异常，本次设置可能无法保存" | 降级为内存存储，刷新后丢失 |
| Service Worker 注册失败 | 不影响核心功能，PWA 安装提示不显示 | 静默失败，用户仍可通过浏览器正常使用 |
| 网络断开 | AI 分析功能不可用，其他功能正常 | AI 分析按钮置灰，提示"需要网络连接" |

## 限制

| 维度 | 说明 |
|------|------|
| 用户规模 | 先给大众验证，单用户模式（无账号体系） |
| 数据存储 | 存本地 IndexedDB，关掉再开数据在。具体存储范围见下表 |

**持久化数据清单**（存 IndexedDB）：

| 数据项 | 存储字段 | 说明 |
|--------|----------|------|
| 节拍器 BPM | `metronome.bpm` | 用户最后设置的 BPM 值 |
| 节拍器音色 | `metronome.sound` | 'beep' / 'drum' / 'voice' |
| 间歇计时模式 | `interval.mode` | 'auto' / 'manual' |
| 间歇计时方案 | `interval.plan` | 预置方案名或 'custom' |
| 自定义训练秒数 | `interval.trainSec` | 仅当 plan='custom' 时 |
| 自定义休息秒数 | `interval.restSec` | 仅当 plan='custom' 时 |
| 自定义组数 | `interval.rounds` | 仅当 plan='custom' 时 |
| 喝水间隔 | `water.interval` | 分钟数 |
| 喝水提醒开关 | `water.enabled` | true/false |
| 运动联动开关 | `water.sportLink` | true/false |

**不持久化的数据**（每次打开 APP 重置）：
- 上次跑步用时（仅当次会话显示，关闭后清空）
- 今日喝水次数（每天自动归零，见统计规则）
- 间歇计时当前进度（关闭 APP 即中断，不恢复）

| 维度 | 说明 |
|------|------|
| 成本 | 核心功能零成本；AI 训练计划解析有少量 API 费用（每次约 ¥0.01-0.05） |
| 技术能力 | 用户不会写代码，全凭 AI 工具开发 |
