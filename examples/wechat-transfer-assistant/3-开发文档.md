# 开发文档（SDD）— 微信转账对账助手 v4

## 1. 概述

- **产品定位**：上传微信聊天截图，OCR 识别并提取转账信息，生成入金/出金对账表，支持编辑和导出 Excel
- **对应 PRD**：`1-PRD.md`
- **本期范围**：截图上传 → OCR → 提取 → 分表展示 → CRUD → 导出 Excel
- **本期不做**：登录、多人协作、支付集成
- **项目元信息**：
  - 模块名：`github.com/user/wechat-transfer-recon`
  - 语言版本：Go ≥ 1.21
  - 端口：从 `PORT` 环境变量读取，默认 `8080`
  - 配置格式：`.env` 文件 + 环境变量覆盖

## 2. 总体架构

```
浏览器 ←→ Gin (Go) ←→ PostgreSQL
                ↕
         百度云 OCR API (HTTP)
```

前端通过 Go embed 内嵌在二进制中，同域部署。

## 3. 端到端流程

| 步骤 | 角色 | 实现 | 说明 |
|------|------|------|------|
| ① 打开页面 | User | `web/index.html` | 加载页面 |
| ② 上传截图 | User | `handler/upload.go` POST /api/upload | 选择图片 |

| ⑥ 返回 | System | → JSON | records+total+duplicated |

## 3b. 外部服务集成：百度云 OCR

### 调用方式：HTTP REST（不用 SDK）

百度云 OCR Go SDK 多年未更新，**直接 HTTP POST 调 REST API**。

### 获取 access_token

```
POST https://aip.baidubce.com/oauth/2.0/token
参数：
  grant_type=client_credentials
  client_id={BAIDU_OCR_API_KEY}
  client_secret={BAIDU_OCR_SECRET_KEY}
响应：{ "access_token": "xxx", "expires_in": 2592000 }
```

access_token 有效期 30 天，缓存到变量中复用，过期后重新获取。

### 文字识别请求

```
POST https://aip.baidubce.com/rest/2.0/ocr/v1/general_basic?access_token={token}
Content-Type: application/x-www-form-urlencoded
请求体：image={base64编码图片}
响应：{
  "words_result": [
    { "words": "2024年6月1日 14:30" },
    { "words": "张三" },
    { "words": "向你转账¥200.00" }
  ],
  "words_result_num": 3,
  "log_id": 123456789
}
```

### 限流与错误处理
- 百度 OCR 免费额度：500次/天，2 QPS
- 后台加简单限流：每次 OCR 请求后 sleep 500ms
- 返回错误码时：记录日志，返回前端"识别失败，请稍后重试"
- 图片调 OCR 前压缩：最长边 ≤ 1920px，文件 ≤ 4MB

## 6. 关键数据结构 / 接口约定

### 数据模型

```go
type Record struct {
    ID         int64   `json:"id"`
    CreatedAt  string  `json:"created_at"`
    TxTime     string  `json:"tx_time"`      // "YYYY-MM-DD HH:MM"
    Party      string  `json:"party"`        // 空字符串 = 未知
    Direction  string  `json:"direction"`    // "入金"|"出金"
    Amount     float64 `json:"amount"`       // ¥, 两位小数
    Note       string  `json:"note"`
    SourceFile string  `json:"source_file"`
    DedupHash  string  `json:"-"`
}
```

### 数据库 DDL

```sql
CREATE TABLE IF NOT EXISTS records (
    id          SERIAL PRIMARY KEY,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    tx_time     VARCHAR(64) NOT NULL DEFAULT '',
    party       VARCHAR(128) NOT NULL DEFAULT '',
    direction   VARCHAR(8) NOT NULL DEFAULT '' CHECK (direction IN ('入金', '出金', '')),
    amount      DECIMAL(12,2) NOT NULL DEFAULT 0 CHECK (amount >= 0),
    note        TEXT DEFAULT '',
    source_file VARCHAR(256) DEFAULT '',
    dedup_hash  VARCHAR(64) NOT NULL DEFAULT '',
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX IF NOT EXISTS idx_records_dedup ON records(dedup_hash) WHERE dedup_hash != '';
CREATE INDEX IF NOT EXISTS idx_records_direction ON records(direction);
CREATE INDEX IF NOT EXISTS idx_records_tx_time ON records(tx_time DESC);
-- 程序启动时执行以上 DDL，无需人工操作数据库
```

### REST API

**错误格式**：`{ "error": "信息", "code": 400 }`
**命名**：全小写 snake_case

#### POST /api/upload
请求：`multipart/form-data { file: 图片 }`（PNG/JPG, ≤10MB）
响应 200：
```json
{"records":[{"id":1,"tx_time":"2024-06-01 14:30","party":"张三","direction":"入金","amount":200.00,"note":""}],"total":5,"duplicated":0}
```
响应 400：`{"error":"仅支持 PNG 和 JPG 格式","code":400}`

#### GET /api/records?direction=in
响应：`{"records":[...],"total":5}`

#### POST /api/records（手动添加）
请求：`{"tx_time":"...","party":"...","direction":"入金","amount":200.00,"note":""}`
校验：tx_time 必填、party 非空、direction ∈ {入金,出金}、amount ≥ 0.01
响应 201：`{"record":{"id":42,...}}`

#### PUT /api/records/:id
请求：同上（全量更新）
**实现要点**：更新后同步重新计算 `dedup_hash`，更新 `updated_at`
前端行为：保存后刷新两个 Tab 的数据+统计；如果 direction 变了则刷新两个 Tab，否则只刷当前 Tab

#### DELETE /api/records/:id
响应 200：`{"ok":true}`
前端：先弹确认窗"确定删除此记录？"→ 确认后发 DELETE

#### GET /api/export?direction=
direction 可选（空=全部两个 sheet）
响应：Content-Type application/vnd... → 文件名 `转账对账_全部_2026-06.xlsx`

### 核心算法：OCR 文本解析引擎

#### 输入示例
```json
{"words_result":[{"words":"2024年6月1日 14:30"},{"words":"张三"},{"words":"向你转账¥200.00"}]}
```

#### ParseRecords(words_result) → []Record
```
逐行扫描 words_result，遇到空行或时间戳行切分消息块
每块调 parseBlock()
```

#### parseBlock(lines) → Record or nil
```
1. 排除：含系统通知/已退还/已过期 → nil
2. 排除：含"红包"且不含"转账" → nil
3. 判定方向：
   入金关键词命中 → direction="入金"
   出金关键词命中 → direction="出金"
   都不命中 → nil
4. 提取金额：
   regex ¥(\d{1,3}(,\d{3})*\.\d{2}) → parseFloat(去千分位)
   或 regex (\d+\.\d{2})元 → parseFloat
   都匹配不到 → nil
5. 提取时间 → extractTime(fullText)
6. 提取交易方 → extractParty(lines, direction)
7. dedupHash = SHA256(tx_time+"|"+party+"|"+fmt.Sprintf("%.2f", amount))
```

#### extractTime(fullText) → string
```
m = regex: (\d{4})[年-](\d{1,2})[月-](\d{1,2})[日\s]+(\d{1,2}):(\d{2})
  → year + "-" + pad2(month) + "-" + pad2(day) + " " + pad2(hour) + ":" + min
无时间部分 → 补 "00:00"
pad2(s): len==1 → "0"+s, else s
```

#### extractParty(lines, direction) → string
```
出金：regex "你向(.+?)转账" → group[1]
入金：时间行之后、首条关键词行之前，第一行非空非关键词行
都找不到 → ""
```

## 8. 部署（Railway）

1. 注册 Railway → GitHub 登录
2. 创建项目 → 选 GitHub 仓库
3. Railway 自动检测 Go 项目（从 `go.mod` 读 Go 版本）
4. 无需 Procfile
5. 添加 PostgreSQL 插件 → 自动生成 `DATABASE_URL`
6. 设置环境变量：`BAIDU_OCR_API_KEY` `BAIDU_OCR_SECRET_KEY`
7. 部署 → Railway `go build` → 分配 URL
8. **端口**：从 `PORT` 环境变量读取，Railway 自动分配
9. **健康检查**：`GET /api/health`
10. 验证：打开 URL → 上传截图测试

## 9. 安全底线

- 密钥仅通过环境变量传递，不写代码、不进前端
- 上传文件校验：仅允许 PNG/JPG（MIME + 文件头魔数），≤ 10MB
- 参数化查询：禁止字符串拼接 SQL
- API 输入校验：direction ∈ {入金,出金}、amount ≥ 0.01
- 错误信息不泄露内部细节（栈追踪、数据库错误等）
- HTTPS 由 Railway 自动配置
