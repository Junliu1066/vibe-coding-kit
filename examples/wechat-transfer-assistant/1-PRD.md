# 产品需求文档（PRD）— 微信转账对账助手 v4

## 一句话

上传微信聊天记录截图，自动识别并提取其中的转账信息，生成入金/出金两张对账表，支持查看、编辑和导出 Excel。

## 背景与问题

小微商户、自由职业者和有频繁转账往来的个人，月底对账时面临重复性痛点：需要一条条翻微信聊天记录，手动找到每一笔转账信息，再复制粘贴到 Excel。一次对账往往要处理几十上百条记录，耗时 1-2 小时，容易遗漏或抄错。市面上没有现成产品能自动从截图中提取转账信息。

## 目标用户与场景

- **主要用户**：有小额高频转账对账需求的人（小微店主、自由职业者、代购、群收款管理者）
- **使用场景**：月底集中对账时上传聊天截图 → 系统自动提取 → 编辑确认 → 导出 Excel
- **使用设备**：手机（拍/选截图）和电脑（查看编辑导出）

## 产品目标与成功指标

- **目标**：将对账效率从手工 1-2 小时降低到 5-10 分钟
- **成功指标**：OCR 金额字段准确率 ≥ 90%；10 个真实用户试用多数反馈"好用"；单次对账操作 ≤ 4 步

## UI/交互说明

### 整体布局（从上到下 7 层）
1. **标题栏**：产品名称 + 简短描述
2. **上传区域**：居中虚线边框拖拽区，📁 图标 + "点击上传截图" + "支持 PNG/JPG"
3. **统计卡片行**：4 张横排卡片——「入金总计」「出金总计」「总记录数」「去重过滤」
4. **Tab 栏**：两个 Tab——「📥 入金表」「📤 出金表」，激活态下划线
5. **操作按钮行**：「＋手动添加」（左）「📥导出 Excel」（右）
6. **数据表格**：列「时间」「交易方」「金额」「备注」「操作」，✏️ 🗑
7. **底部**：原型标注

### 状态设计
- **空态**：无数据时，上传区下方显示"上传一张截图开始对账"引导文字 + 示例截图缩略图，表格显示"暂无记录"
- **加载态**：上传中→进度环"正在识别..."；导出中→按钮变"导出中..."禁用
- **错误态**：OCR 失败→红色边框+"识别失败，请重试"；网络断→顶部黄色横幅"网络连接异常"；格式不对→"仅支持 PNG 和 JPG"
- **编辑态**：点击 ✏️ → 居中模态框，5 个字段（时间/交易方/方向【下拉框】/金额/备注），保存后刷新数据+统计
- **移动端（≤768px）**：统计卡片纵向堆叠，表格横向滚动

## 核心功能

1. [P0] **截图上传**：PNG/JPG，单张，自动触发识别
2. [P0] **OCR 识别提取**：百度云 OCR → 按规则提取转账记录
3. [P0] **出入金分表展示**：入金/出金两张表 Tab 切换
4. [P0] **手动添加**：弹窗填时间/交易方/方向（下拉框）/金额/备注
5. [P0] **手动修改**：弹窗编辑，保存后刷新
6. [P0] **导出 Excel**：两个 sheet，金额格式 `¥1,200.00`
7. [P1] **去重**：tx_time+party+amount 去重
8. [P1] **首次引导**：空态显示示例
9. [P1] **删除确认**：弹窗"确定删除？"

## 核心功能的处理规则

### 提取规则（正则级）

| 目标 | 正则 | 示例 |
|------|------|------|
| 时间 | `\d{4}[年-]\d{1,2}[月-]\d{1,2}[日\s]+\d{1,2}:\d{2}` | `2024年6月1日 14:30` |
| 入金关键词 | `向你转账\|收到转账\|已收款\|收款[¥￥]\|入账[¥￥]` | `向你转账¥200.00` |
| 出金关键词 | `你向.+转账\|转出[¥￥]\|付款[¥￥]\|支出[¥￥]` | `你向王五转账¥1,200.00` |
| 金额（¥前缀） | `[¥￥](\d{1,3}(,\d{3})*\.\d{2})` | `¥1,200.00` → 1200.00 |
| 金额（元后缀） | `(\d+\.\d{2})元` | `200.00元` → 200.00 |

**交易方提取**：入金=取关键词前一行；出金=`你向(.+?)转账`
**排除项**：系统通知、已退还、已过期、红包（不含"转账"）
**缺字段兜底**：时间/金额/方向缺→跳过；交易方缺→存空字符串，前端显示`——`

### 去重
条件：`tx_time + party + amount`（时间精确到分钟，金额两位小数）
策略：`INSERT ON CONFLICT (dedup_hash) DO NOTHING`

### 排序
按时间降序（最新最上）

## 出错和边界

| 场景 | 用户看到 | 系统行为 |
|------|---------|---------|
| 非 PNG/JPG 格式 | "仅支持 PNG 和 JPG 格式" | 拒绝，不调 OCR |
| 文件 > 10MB | "图片超过 10MB，请压缩后上传" | 拒绝 |
| OCR 返回空 | "未识别到文字，请检查截图是否清晰" | 返回空列表 |
| 有文字无转账 | "未识别到转账记录，请确认截图包含转账信息" | 返回空列表 |
| 图片模糊致缺字段 | 缺字段显示 `——` | 可提取的写入，缺失的存 `""` |
| 网络超时 30s | 黄色横幅"网络连接异常，请检查后重试" | 中断请求 |
| 重复上传 | "已识别 N 条，X 条重复已过滤" | ON CONFLICT 去重 |
| 金额填非数字 | 红色边框+"请输入有效金额（大于0的数字）" | 不提交 |
| 交易方为空 | "交易方不能为空" | 不提交 |
| 服务端异常 | "服务暂时不可用，请稍后重试" | 500 + 日志 |

1. 上传一张含 5 笔以上转账的清晰截图 → 系统识别出所有记录，按入金/出金分表展示
2. 金额字段准确率 ≥ 90%
3. 可手动添加一条新记录，保存后出现在对应表格
